# MCP Server Specification

This document defines the exact Model Context Protocol (MCP) servers used by Wisdom to interact with local knowledge management and study tools, resolving the gap in implementation details.

## 1. Obsidian MCP Server
*   **Tool:** `obsidian-mcp-server`
*   **Deployment:** Runs locally on the user's machine (e.g., via `npx` or as a background service).
*   **Prerequisites:** Obsidian must have the `Local REST API` plugin installed and running.
*   **Authentication & Security:** 
    *   Communication is restricted to `localhost`.
    *   Requires an API Key generated by the Local REST API plugin, injected via the `OBSIDIAN_API_KEY` environment variable.
*   **Exposed Tools:**
    *   `search_vault(query)`: Semantic and keyword search.
    *   `read_note(path)`: Retrieves note content and frontmatter.
    *   `create_note(path, content)`: Creates new Markdown files.
    *   `update_note(path, content)`: Surgical edits to existing notes.

## 2. Anki MCP Server
*   **Tool:** `anki-mcp-server` (FastMCP implementation).
*   **Deployment:** Runs locally, interfacing with the Anki desktop application.
*   **Prerequisites:** Anki desktop app must be open with the `AnkiConnect` add-on installed.
*   **Authentication & Security:**
    *   AnkiConnect runs on `http://localhost:8765`.
    *   AnkiConnect must be configured to accept CORS requests from the local MCP server.
*   **Exposed Tools:**
    *   `add_note(deck_name, model_name, fields, tags)`
    *   `find_notes(query)`
    *   `get_reviews(query)`: Critical for the Wisdom Sync feedback loop.

## 3. Logseq MCP Server
*   **Tool:** `mcp-logseq`
*   **When to use Logseq vs. Obsidian:**
    *   Obsidian is the default for long-form, document-centric knowledge.
    *   Logseq is used **only if the user explicitly prefers an outliner/block-based graph**.
    *   *Conflict Resolution:* Wisdom enforces directory separation. Logseq pages are kept in a dedicated `logseq-pages/` directory to prevent Obsidian markdown parsers from breaking on EDN or Logseq-specific block attributes.
*   **Authentication:** Requires an API token from Logseq's Local HTTP API settings.
*   **Exposed Tools:**
    *   `logseq_insert_block(parent_id, content)`
    *   `logseq_query(datalog_query)`

## 4. Fallback & Error Handling
If an MCP server is unavailable (e.g., Anki is closed, or the user is on mobile):
1.  **Staging Area:** Wisdom catches the `ECONNREFUSED` error.
2.  **Queueing:** The JSON payload is saved to the `Cortex` database with a status of `PENDING_SYNC`.
3.  **Portal Notification:** The Mission Control Portal alerts the user: *"5 Anki cards pending. Please open Anki to sync."*
4.  **Retry:** The `Integrations` service attempts to sync every 5 minutes until successful.
