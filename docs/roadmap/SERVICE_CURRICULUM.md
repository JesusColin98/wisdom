# Subsystem: Wisdom-Curriculum (Orchestrator)

The Curriculum service is the "Academic Dean" of Wisdom. It transforms raw research into a structured, logical sequence of learning.

## Logic: Big Topics & Sub-topics
Information is organized in a recursive hierarchy:
1.  **Macro-Domain** (e.g., Science, Humanities, Technical)
2.  **Big Topics** (e.g., Natural Sciences, History, Computing)
3.  **Sub-topics** (e.g., Biology, Cyber-security, French History)
4.  **Learning Modules** (e.g., Cell Division, SQL Injection, French Revolution)
5.  **Atomic Concepts** (e.g., Mitosis, Buffer Overflow, Bastille Day)

## Use Cases
- **Path Generation**: Organize a collection of "Researcher" signals into a roadmap.
- **Dependency Mapping**: Determine if Concept A is a prerequisite for Concept B.
- **Leveling**: Assign difficulty tiers (1-5) to modules.

## Pre-defined Taxonomy (Sample)
New topics are generated by matching incoming research to this taxonomy or extending it if similarity is low.

| Macro-Domain | Big Topic | Sub-topics |
| :--- | :--- | :--- |
| **Technical** | Computing | Cloud, Cyber-sec, AI, Web |
| **Creative** | Games | Chess, Go, Game Design |
| **Humanities** | History | Ancient, Modern, Regional |
| **Languages** | Linguistics | English, French, Spanish |

## Interface

```protobuf
service Curriculum {
  rpc GeneratePath(PathRequest) returns (LearningPath);
  rpc MapDependencies(NodeList) returns (DependencyGraph);
}

message PathRequest {
  string topic = 1;
  int32 target_level = 2; // 1: Beginner, 5: Expert
}

message LearningPath {
  string topic_id = 1;
  repeated Module modules = 2;
}
```
