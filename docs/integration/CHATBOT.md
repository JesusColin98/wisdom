# Integration Guide: Project Wisdom Chatbot

This guide explains how to build advanced Chatbots using the Wisdom Neural-Socratic loop.

## 1. The Neural-Socratic Protocol

Unlike standard RAG, Wisdom Chatbots engage in a dialogue with the knowledge substrate:
1. **Analyze Gaps:** Wisdom identifies what it *doesn't* know about the query.
2. **Expand Seeds:** It automatically fetches related nodes to bridge those gaps.
3. **Refine Context:** The final context sent to your LLM is grounded and relational.

## 2. Intent-Driven Retrieval

Your chatbot doesn't need to decide how to search. The Thalamus handles it:
- **"Show me the org chart for X":** Thalamus detects `HIERARCHY` and uses recursive CTEs.
- **"What's the risk of deploying Y?":** Thalamus detects `STAR` and uses the **Risk Engine**.
- **"Debug this 429 error":** Thalamus detects `KG` and uses the **SRE Assistant**.

## 3. Real-time Logit Validation

Integrate with the SCG-Mem Trie via WebSockets. Every sentence generated by your LLM can be cross-referenced against the "World of Known Truths" before the user sees it.

### Warning Injection
If the Thalamus detects a structural hallucination, it can inject a `[STRICT MODE WARNING]` directly into the session stream, allowing your chatbot to self-correct.
